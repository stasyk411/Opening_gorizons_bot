# Библиотека мыслительных узлов

## Принцип

Бот работает не с типами людей, архетипами или диагнозами, а с **мыслительными узлами** — местами, где человек склеил две разные вещи.
Библиотека состоит из 20–30 универсальных узлов. Каждый узел проявляется в тысячах жизненных ситуаций, но механизм ошибки — один.

## Двухуровневая структура

1. **Узел** (например, «Факт = вывод») — универсальный механизм.
2. **Варианты проявления** (например, «Он молчит → Он меня ненавидит») — конкретные человеческие истории.

Бот распознаёт узел, а не историю. Вопросы направлены на разделение склейки, а не на обсуждение сюжета.

---

## Блок 1. Ложные выводы

1. **Факт = вывод** («Он молчит» → «Он меня ненавидит»)
2. **Чувство = факт** («Мне страшно» → «Значит опасно»)
3. **Мысль = реальность** («Мне кажется» → «Так и есть»)
4. **Вероятность = неизбежность** («Это может случиться» → «Это обязательно случится»)

## Блок 2. Узкие варианты

5. **Или / Или** («Или терпеть, или уходить»)
6. **Один путь** («Других вариантов нет»)
7. **Всё рухнет** (Катастрофизация)

## Блок 3. Чужая голова

8. **Чтение мыслей** («Я знаю, что он думает»)
9. **Чтение мотивов** («Он специально»)
10. **Чужое настроение = я** («Он злой» → «Это из-за меня»)

## Блок 4. Я

11. **Ошибка = я плохой** («Я ошибся» → «Я плохой»)
12. **Отказ = меня отвергли**
13. **Критика = меня не любят**

## Блок 5. Отношения

14. **Я отвечаю за чувства другого** («Если он страдает — это моя вина»)
15. **Я должен спасти**
16. **Без меня он пропадёт**
17. **Без него я пропаду**

## Блок 6. Потери

18. **Потеря = конец** («Если уйдёт — жизнь закончится»)
19. **Изменение = опасность**
20. **Неизвестность = плохо**

## Блок 7. Прошлое

21. **Всегда** («Так было» → «Так будет»)
22. **Один случай = закон** («Один раз» → «Всегда»)

## Блок 8. Контроль

23. **Я должен всё контролировать**
24. **Если отпущу — всё разрушится**
25. **Я обязан** («Я должен / Я обязан / Я не имею права»)

## Именование файлов в коде

Не использовать диагностические термины (`scene_catastrophizing.md`).
Не использовать абстрактные коды (`scene_bridge_001.md`).
Использовать человеческие формулировки, отражающие суть склейки:

- `scene_fact_equals_conclusion.md`
- `scene_i_know_what_he_thinks.md`
- `scene_everything_will_collapse.md`
- `scene_i_must.md`

Цель: при открытии папки сразу понимать, какой узел обрабатывает сцена, без перевода с психологического языка.
