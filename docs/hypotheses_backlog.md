# Hypotheses Backlog

## Назначение

Этот документ фиксирует все новые идеи как проверяемые предположения, а не как части ядра. Никакая функциональность не добавляется в V1 без прохождения цикла «гипотеза → проверка → решение».

## Метрики успеха V1 (Нулевые вопросы)

1. Люди понимают первый экран? (конверсия в нажатие кнопки)
2. Они готовы написать после выбора EP? (конверсия в первое сообщение)
3. Им становится легче после разговора? (субъективная оценка / завершение сцены)
4. Они делают следующий шаг? (подтверждение микрошага)
5. Они возвращаются? (ретентивность через 7 дней)

## Активные гипотезы

### Гипотеза №1

**Суть:** Если перед переходом к сцене спросить «Что сейчас для тебя самое важное?», то средняя длина диалога увеличится.
**Метрика:** Количество сообщений до завершения сцены.
**Критерий успеха:** Увеличение медианной длины диалога на ≥20% без роста оттока.
**Статус:** Новая
**Решение:** —

### Гипотеза №2

**Суть:** Кнопка «🚨 Срочно нужна помощь» на первом экране снижает количество кризисных обращений в свободный ввод.
**Метрика:** Доля экстренных запросов через EP vs свободный текст.
**Критерий успеха:** ≥70% кризисных запросов идут через кнопку.
**Статус:** Новая
**Решение:** —

### Гипотеза №3

**Суть:** Если бот теряет хронологию событий, пользователь тратит ресурс на восстановление контекста вместо проработки проблемы.
**Метрика:** Количество сообщений-уточнений от пользователя со словами «вчера/сегодня», «ты путаешь», «повторяю».
**Критерий успеха:** 0 случаев потери хронологии в диалогах длиннее 10 сообщений.
**Статус:** Новая
**Решение:** —

### Гипотеза №4

**Суть:** Экстренный режим срабатывает ложно при отсутствии острых маркеров, включая контекст выбранной EP и упоминание слов вроде «тревога» в исследовательском контексте.
**Метрика:** Количество срабатываний экстренного скрипта в диалогах, где пользователь явно в ресурсе (мечтает, анализирует, уточняет) и отрицает острое состояние.
**Критерий успеха:** 0 ложных срабатываний при отсутствии явных маркеров угрозы и наличии отрицания пользователем.
**Статус:** В тесте
**Решение:** Доработать

### Гипотеза №5

**Суть:** При запросе «переформулируй проще» бот продолжает выдавать метафоры и длинные объяснения вместо коротких бытовых фраз.
**Метрика:** Количество сообщений после запроса «проще», содержащих более 3 предложений или абстрактные метафоры.
**Критерий успеха:** Ответ после «проще» ≤ 2 предложений и без метафор в ≥80% случаев.
**Статус:** Новая
**Решение:** —

### Гипотеза №6

**Суть:** Safety Layer интерпретирует выбор категории (например, «Мне тревожно») как подтверждение острого состояния даже при отсутствии маркеров в тексте, что приводит к превентивному включению кризисного протокола.
**Метрика:** Доля экстренных ответов, сработавших исключительно на основе EP без совпадения с PANIC_MARKERS/SUICIDE_MARKERS.
**Критерий успеха:** 0 превентивных срабатываний; экстренный режим активируется только при наличии явных текстовых маркеров.
**Статус:** Новая
**Решение:** —
