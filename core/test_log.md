## Тест 05: Длинный диалог с перегрузкой и инверсией ролей

**Дата:** 28.06.2026
**Контекст:** Пользователь в состоянии острого стресса («туман», «трясёт»), муж уходит, финансовые проблемы.
**Наблюдения:**

1. Роутер работает корректно: safety=False, structural=None, сцены активируются (03, 06, 01, 02).
2. При фразе «меня трясёт» / «в тумане» confidence падает до 0.3–0.6 → unknown → бот скатывается в дефолтную стабилизацию. Нет сцены для состояния перегрузки.
3. LLM дважды инвертировала роли: «дети отвечают за тебя» вместо «ты отвечаешь за детей». Пользователь исправил, бот самокорректировался.
4. Несмотря на ошибки, диалог привёл к реальному действию (звонок арендодателю).
   **Выводы:**

- Нужна сцена `scene_07_overload.md` для состояний «туман/трясёт/не могу думать».
- В системный промпт добавить правило: «Проверяй субъект ответственности. Никогда не инвертируй роли (родитель↔ребёнок). При сомнении — спрашивай.»
- Гибкость бота (самокоррекция) важнее идеального соблюдения формата.
